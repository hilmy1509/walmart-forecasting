/*
=========================================================================
ARCHITECTURAL ANALYSIS & FORENSIC CODE BREAKDOWN:
=========================================================================
This script represents the direct atomic query compiled by the Power BI 
engine to compute and render the primary single-value KPI metric card 
representing the total aggregated future sales projection.

Key Mechanics Observed:
1. THE ROW() FUNCTION SEED:
   - Unlike complex graphical trends that utilize multi-column cross-joins 
     (`SUMMARIZECOLUMNS`), single-value visual objects (like KPI Cards) 
     leverage the native `ROW()` constructor.
   - `ROW()` generates a virtual single-row, single-column scalar metadata 
     matrix containing exactly one key-value payload entry. This minimizes 
     engine computation costs, forcing a fast layout render.

2. CALCULATE CONTEXT CONVERTER:
   - The expression explicitly wraps the `SUM()` aggregation inside a 
     `CALCULATE()` block. 
   - This architecture is critical: it guarantees that any incoming filter 
     context generated by slicers (e.g., selecting specific Store/Dept from 
     the `Dim_Store` or `Dim_Dept` dimensions) will cleanly propagate down 
     and filter the final dataset before the calculation is frozen into the 
     KPI display text.
*/

// Step 1: Execute immediate scalar row transmission
EVALUATE
    ROW(
        "SumForecasted_Sales", 
        CALCULATE(
            SUM('walmart_sales_predictions_murni_future'[Forecasted_Sales])
        )
    )
